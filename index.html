DATA GENERATION INSTRUCTIONS
Use this checklist before loading a trace

Generate a Chrono-Matrix trace in three steps

Instrument your C++ workload, run it with the Chrono-Matrix tracer, then load the generated JSON file here. Each event should include a timestamp, thread id, event type, duration, and optional resource/address context.

1Annotate work

Use hooks like CM_COMPUTE("phase") and CM_MEM_ACCESS(ptr, size, "MEM_READ", "phase").

2Run under tracer

Build tracer/libcmhijack.so, then run your binary with LD_PRELOAD.

3Load JSON

Drop the generated program-name.json file here or use the top-bar Load Trace button.

#
COMMAND TEMPLATE
From project root 
g++ -std=c++17 -O2 -fPIC -shared tracer/cm_hijacker.cpp \
  -o tracer/libcmhijack.so -ldl -pthread

g++ -std=c++17 -O2 -pthread tracer/producer_consumer.cpp \
  -o tracer/producer_consumer -ldl

LD_PRELOAD=./tracer/libcmhijack.so ./tracer/producer_consumer
{ }
EXPECTED JSON SHAPE
Array of events 
[
  {
    "ts": 123456,
    "tid": 4,
    "event": "LOCK_WAIT",
    "duration_us": 820,
    "resource": "mutex_A",
    "addr": "0xAA001000",
    "size": 64,
    "scenario": "producer_consumer"
  }
]
✓
QUALITY CHECKS
Best results need these fields
Required: ts, tid, event
Recommended: duration_us, scenario, resource
Memory views: include addr and size for memory events
Contention views: include stable lock names in resource
